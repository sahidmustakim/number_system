4. Base Finder

Solve base finding problems like "(12)r + (13)r = (25)r":

Results:

r > 5
Step 1: Convert each number to base 10.
(12)r = 1×r + 2 = r + 2
(13)r = 1×r + 3 = r + 3
(25)r = 2×r + 5 = 2r + 5

Step 2: Set up the equation in base 10.
(r + 2) + (r + 3) = 2r + 5

Step 3: Simplify both sides.
Left side: r + 2 + r + 3 = 2r + 5
Right side: 2r + 5
So: 2r + 5 = 2r + 5

Step 4: Solve
Since both sides are equal for any r, the equation holds for all values of r.

Step 5: Apply base constraints
In base r, all digits must be less than r.
From the digits: 1, 2, 3, 5 — the largest digit is 5, so: r > 5
Also, base must be an integer greater than 1: r ∈ ℕ and r > 5

✅ Final Answer:
r can be any integer such that r > 5

5. Handling Corner Cases

Case 1: Multiple Valid Bases

For equations like (12)r + (13)r = (25)r, multiple bases may be valid:

Input: (12)r + (13)r = (25)r
Output: r > 5
Explanation: The equation holds for all bases greater than 5 because the largest digit is 5.

Case 2: Single Valid Base

For equations like (12)r + (13)r = (30)r, only one base may be valid:

Input: (12)r + (13)r = (30)r
Output: r = 5
Explanation: Solving gives r = 5, and verification shows (12)₅ + (13)₅ = 7 + 8 = 15 = (30)₅.

Case 3: No Valid Base

Some equations have no solution:

Input: (12)r = (5)r
Output: No solution
Explanation: This would require 1×r + 2 = 5, so r = 3, but base 3 cannot have digit '5'.

Case 4: Fractional Results

For division operations that result in fractions:

Input: (10)r ÷ (2)r
Output: 5 (for r > 5)
Explanation: In bases where both numbers are valid (r > 5), this equals 5 in decimal.
